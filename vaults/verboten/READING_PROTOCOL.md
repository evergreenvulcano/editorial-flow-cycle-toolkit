# Protected Reading Protocol

This protocol defines how `verboten.pdf` may be read by humans or AI-assisted tools inside the vault.

## Stage 0 — No-reading foundation

Install vault structure.
Declare source status.
Declare forbidden transformations.
Prepare runlog.

## Stage 1 — Sample reading

Read a small bounded excerpt.
Produce only structural notes.
No voice imitation.
No derivative prose.
No public-facing language.

## Stage 2 — Full pass candidate

Only after review of Stage 1.
Define exact reading aim.
Define output format.
Define reviewer.
Define stop conditions.

## Stage 3 — Editorial map

Map motifs, structures, recurring pressures, apparatus logic, and unresolved editorial affordances.

## Stage 4 — Content Desk gate

Only installed if the editorial team explicitly decides that some material may become public-facing content.

Until then, the work remains protected source material.
