# Verboten Vault

This vault is the protected workspace around `verboten.pdf`.

It defines process, status, and review constraints before any AI-assisted reading occurs.

## Vault rule

- Source work stays outside git.
- Repository records protocol, provenance, and status only.
- No generated literary output is produced in this foundation branch.

## Vault files

- `STATUS.md` — current vault and material status
- `SOURCE_MANIFEST.example.md` — example manifest format, no source included
- `READING_PROTOCOL.md` — staged reading constraints
- `AI_AUGMENTATION_GUARDRAILS.md` — allowed and forbidden AI modes
- `CONTENT_DESK_GATE.md` — explicit definition of absent-by-default content desk
- `RUNLOG.md` — run record format for future controlled runs
