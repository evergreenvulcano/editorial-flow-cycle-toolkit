# Protected Editorial Vault Foundation

This repository establishes a protected editorial filesystem environment around a literary source file named `verboten.pdf`.

`verboten.pdf` is the source work.
The product is not the source work.
The product is the protected filesystem contract that governs how reading, AI augmentation, review, and output status are handled.

## Core policy

- The source work is not treated as content by default.
- AI augmentation is allowed only through documented protocols, review gates, provenance notes, and explicit output status labels.
- No generated output may inherit, replace, flatten, summarize away, imitate, or commercialize the source voice.
- No source file is committed in this branch.
- A Content Desk gate is absent by default and must be explicitly installed later.
- For new protected-source branches, follow [`docs/SOURCE_VAULT_BRANCH_PROTOCOL.md`](docs/SOURCE_VAULT_BRANCH_PROTOCOL.md).

## Branch scope

This first reviewable branch only installs the minimum vault contract:

- product definition
- review checklist
- vault status
- source manifest example
- reading protocol
- AI guardrails
- content desk gate definition
- runlog format

No ingestion, transformation, or literary output is produced here.
